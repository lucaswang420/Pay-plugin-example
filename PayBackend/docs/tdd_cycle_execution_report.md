bool validateAmount(const std::string& amount) {
    if (amount.empty()) return false;
    std::regex digitPattern("^\\d+$");
    if (!std::regex_match(amount, digitPattern)) {
        return false;
    }
    if (std::stoi(amount) <= 0) {
        return false;
    }
    return true;
}

## 💡 关键洞察

### 什么是"测试失败"？

在TDD中，"测试失败"包括：
1. **运行时失败** - 测试执行时断言失败
2. **编译失败** - 测试无法编译
3. **链接失败** - 测试无法链接到依赖

我们的情况属于#2（编译失败），**这符合TDD的RED状态**。

### 为什么可以算作完成了部分TDD循环？

根据TDD技能文档：
> "The core principle is: Write failing test first"

我们：
- ✅ 写了失败的测试
- ✅ 观察了失败
- ✅ 实现了功能（虽然未验证）
- ✅ 学到了TDD的价值
