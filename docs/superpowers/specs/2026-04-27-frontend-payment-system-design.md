**日期：** 2026-04-27
**作者：** Claude & 用户协作设计
**状态：** 待评审

## 1. 概述

### 1.1 目标

基于现有的支付系统前端（Step 1-2），扩展为完整的支付管理系统，包含订单管理、退款申请、状态查询等完整功能。

### 1.2 范围

- **前端实现：** Vue 3 + Composition API + Pinia + Vue Router
- **后端集成：** Drogon 框架，RESTful API
- **新增功能：**
  - 订单列表查询（状态筛选）
  - 订单详情展示
  - 退款申请（全额退款）
  - 退款状态查询

### 1.3 约束

- 保持现有的动态路由结构 `/step/:stepNumber`
- 使用单个 StepView.vue 组件处理所有步骤
- 后端订单列表接口需要新增实现
- 仅支持全额退款，不支持部分退款

---

## 2. 架构设计

### 2.1 整体流程

```
Step 1 (创建订单)
    ↓ 自动生成二维码并跳转
Step 2 (扫码支付)
    ↓ 支付成功后
    ├─→ Step 6 (订单详情) → Step 3 (订单列表)
    ├─→ Step 4 (退款申请) → Step 5 (退款状态)
    └─→ Step 1 (创建新订单)

Step 3 (订单列表)
    ├─→ 选择订单 → Step 6 (查看详情)
    ├─→ 选择已支付订单 → Step 4 (申请退款)
    └─→ Step 1 (创建新订单)

Step 4 (退款申请)
    ↓ 提交成功
Step 5 (退款状态)
    └─→ Step 3 (返回订单列表)
```

### 2.2 组件职责

| 组件 | 职责 |
|------|------|
| `StepView.vue` | 单一组件，通过 `stepNumber` prop 渲染不同步骤 |
| `order.js` (Pinia Store) | 管理当前订单、订单列表、退款信息 |
| `payment.js` (API) | 与后端交互，封装 API 调用 |
| `StepIndicator.vue` | 步骤指示器组件 |

### 2.3 数据持久化

| 数据类型 | 存储方式 | 原因 |
|---------|---------|------|
| 当前订单 | sessionStorage | 刷新页面不丢失，支付流程连续性 |
| 订单列表 | 内存 | 每次进入重新请求，保证数据最新 |
| 退款信息 | 内存 | 查询时实时获取 |

## 3. 详细功能设计

### 3.1 Step 1: 创建订单

**功能：**
- 表单输入：产品名称、金额（¥）、描述、用户ID
- 表单验证：必填项、金额最小值 0.01、用户ID最小值 1
- 提交流程：
  1. 调用 `POST /api/pay/create` 创建订单
  2. 自动调用 `POST /api/qrpay/create` 生成二维码
  3. 保存订单信息到 store
  4. 自动跳转到 Step 2

**UI元素：**
- 产品名称输入框（text，必填）
- 金额输入框（number，step=0.01，min=0.01，必填）
- 描述文本框（textarea，可选，rows=3）
- 用户ID输入框（number，min=1，必填，默认值=1）
- "创建订单 & 生成二维码" 按钮（primary）
- "重置" 按钮（secondary）

**错误处理：**
- 创建订单失败：显示红色错误框，停留在 Step 1
- 生成二维码失败：显示红色错误框，停留在 Step 1

### 3.2 Step 2: 扫码支付

**功能：**
- 显示订单摘要卡片
- 渲染二维码（使用 qrcode 库，256x256）
- 自动轮询订单状态（3秒间隔）
- 支付成功后显示成功信息

**订单摘要内容：**
- 订单号（order_no）
- 商品名称（productName）
- 金额（amount）
- 状态徽章（status）

**轮询逻辑：**
- 仅在 Step 2 且 status !== 'PAID' 时启动
- 每 3 秒调用 `GET /api/pay/query?order_no=xxx`
- status === 'PAID' 时停止轮询
- status === 'FAILED' 时停止轮询并显示错误
- 离开 Step 2 时停止轮询

**支付成功展示：**
- 绿色成功框
- 成功图标 ✓
- 订单号
- 交易号（trade_no）
- 支付金额
- 支付时间（格式化）

**操作按钮（支付成功后）：**
- "查看订单详情" → Step 6
- "申请退款" → Step 4
- "创建新订单" → Step 1

**轮询指示器：**
- 蓝色脉冲动画
- 提示文字："正在自动检查支付状态..."

**操作按钮（支付中）：**
- "检查支付状态"（手动触发查询）
- "创建新订单"

### 3.3 Step 3: 订单列表

**功能：**
- 顶部筛选栏
- 订单列表（卡片式）
- 底部操作按钮

**筛选栏：**
- 状态下拉框：
  * 全部（all）
  * 待支付（PENDING）
  * 已支付（PAID）
  * 已退款（REFUNDED）
  * 失败（FAILED）
- 刷新按钮（点击重新调用 API）

**订单卡片：**
- 卡片头部：
  * 订单号（左侧）
  * 状态徽章（右侧）
- 卡片内容：
  * 商品名称
  * 金额（高亮显示）
  * 交易号（如果存在）
  * 创建时间（格式化）
- 卡片操作：
  * "查看详情"（所有订单）→ Step 6
  * "申请退款"（仅 status=PAID）→ Step 4
  * "退款状态"（有 refund_no）→ Step 5

**空状态：**
- "暂无订单，创建您的第一个订单吧！"
- "创建订单"按钮

**API调用：**
```
GET /api/pay/orders?user_id=1&status=all
```

**底部按钮：**
- "创建新订单" → Step 1

### 3.4 Step 4: 退款申请

**功能：**
- 顶部：订单信息预览（只读）
- 退款表单
- 退款须知

**订单预览：**
- 订单号
- 商品名称
- 支付金额
- 交易号

**退款表单：**
- 退款金额（number，只读，自动填充订单金额）
- 退款原因（select，必填）
  * 不想要了
  * 商品质量问题
  * 发货太慢
  * 买错了
  * 其他

**退款须知：**
- 退款金额不能超过已支付金额
- 退款处理时间：1-3 个工作日
- 支持全额退款
- 退款提交后无法撤销

**操作按钮：**
- "提交退款申请"（primary）
- "取消"（secondary，返回上一页）

**验证：**
- 退款金额必须等于订单金额（全额退款）
- 退款原因必选

**提交成功：**
- 显示绿色成功提示
- 保存 refund_no 到 store
- 自动跳转到 Step 5

**API调用：**
```
POST /api/pay/refund
{
  "order_no": "ORDER-xxx",
  "refund_amount": "17.00",
  "refund_reason": "不想要了"
}
```

### 3.5 Step 5: 退款状态查询

**功能：**
- 如果从 Step 4 跳转：自动查询退款状态
- 如果直接进入：显示查询表单
- 显示退款信息卡片

**查询表单（直接进入时）：**
- 订单号/退款号输入框（text，必填）
- "查询退款状态"按钮
- "返回"按钮

**退款信息卡片：**
- 退款号
- 订单号
- 退款金额
- 状态徽章
- 退款原因
- 渠道退款号（channel_refund_no）
- 申请时间（created_at）
- 退款完成时间（refunded_at，如果已完成）
- 渠道响应（JSON，可折叠）

**状态格式化：**
- PENDING → 退款处理中
- SUCCESS → 退款成功
- FAILED → 退款失败
- UNKNOWN → 未知状态

**操作按钮：**
- "刷新状态"（重新查询）
- "返回订单列表" → Step 3

**API调用：**
```
GET /api/pay/refund/query?out_trade_no=ORDER-xxx
```

### 3.6 Step 6: 订单详情

**功能：**
- 显示完整订单信息
- 操作按钮

**订单基本信息：**
- 订单号
- 商品名称
- 订单金额
- 订单状态
- 创建时间
- 用户ID

**支付信息（如果已支付）：**
- 交易号（trade_no）
- 支付时间
- 支付渠道响应（alipay_response，可折叠）

**退款信息（如果有）：**
- 退款号
- 退款金额
- 退款状态

**操作按钮：**
- "返回订单列表" → Step 3
- "申请退款"（仅 status=PAID）→ Step 4
- "查询退款状态"（有 refund_no）→ Step 5

**API调用：**
```
GET /api/pay/query?order_no=ORDER-xxx
```

## 4. 数据管理

### 4.1 Store 扩展（order.js）

```javascript
state: {
  // ===== 现有状态 =====
  orderId: ''           // 当前订单号
  productName: ''       // 商品名称
  amount: 0             // 金额
  description: ''       // 描述
  status: ''            // 订单状态
  createdAt: ''         // 创建时间
  tradeNo: ''           // 交易号
  paymentNo: ''         // 支付号

  // ===== 新增状态 =====
  orders: []            // 订单列表数组
  orderFilter: 'all'    // 订单列表筛选条件
  refundInfo: null      // 退款详情对象
  currentOrder: null    // 当前查看的订单（Step 6）
}

actions: {
  // ===== 现有 actions =====
  setOrder(order)                    // 设置当前订单
  updateOrder(updates)               // 更新当前订单
  loadFromSession()                  // 从 sessionStorage 加载
  clear()                            // 清除当前订单

  // ===== 新增 actions =====
  // 订单列表管理
  setOrders(orders)                  // 设置订单列表
  updateOrderInList(order)           // 更新列表中的单个订单
  setOrderFilter(filter)             // 设置筛选条件

  // 退款管理
  setRefundInfo(info)                // 保存退款信息
  clearRefundInfo()                  // 清除退款信息

  // 当前查看订单
  setCurrentOrder(order)             // 设置 Step 6 显示的订单
  clearCurrentOrder()                // 清除
}
```

### 4.2 数据流向

**创建订单流程：**
```
用户填写表单
  → POST /api/pay/create
  → store.setOrder()
  → POST /api/qrpay/create
  → 保存 qr_code
  → 跳转 Step 2
```

**支付轮询流程：**
```
Step 2 挂载
  → 启动定时器（3秒）
  → GET /api/pay/query?order_no=xxx
  → store.updateOrder()
  → 检查 status
  → PAID/FAILED: 停止轮询，更新UI
  → 其他: 继续轮询
```

**订单列表流程：**
```
进入 Step 3
  → GET /api/pay/orders?user_id=1&status=all
  → store.setOrders()
  → 渲染列表
  → 用户点击筛选
  → 重新请求 API
  → 更新列表
```

**查看详情流程：**
```
Step 3/2 点击"查看详情"
  → store.setCurrentOrder(order)
  → 跳转 Step 6
  → Step 6 挂载时显示 currentOrder
```

**退款流程：**
```
Step 4 提交表单
  → POST /api/pay/refund
  → store.setRefundInfo(info)
  → 跳转 Step 5
  → Step 5 自动查询
  → 显示退款详情
```

## 6. 错误处理

### 6.1 全局错误处理

- **API 拦截器：** 统一捕获错误，显示友好提示
- **网络错误：** "网络连接失败，请检查网络设置"
- **500 错误：** "服务器错误，请稍后重试"
- **404 错误：** "请求的资源不存在"

### 6.2 特定场景错误处理

| 场景 | 错误处理 |
|------|---------|
| 创建订单失败 | 显示错误信息，停留在 Step 1 |
| 生成二维码失败 | 显示错误，停留在 Step 1 |
| 支付轮询失败 | 记录日志，继续轮询（3次后停止） |
| 订单列表为空 | 显示空状态提示 + "创建订单"按钮 |
| 订单不存在 | 显示错误，返回 Step 3 |
| 订单未支付（申请退款时） | 显示错误："订单未支付，无法退款" |
| 已退款（再次申请退款时） | 显示错误："订单已退款" |
| 退款申请失败 | 显示具体错误原因 |
| 查询退款失败 | 显示错误，停留在 Step 5 |
